# 0008 Use uv for Python and dependency management

- Date: 2026-03-03
- Author(s): [Jonathan Moss][jmoss]
- Status: `Active`

## Decision

Use [uv][uv] as the single tool for Python installation, dependency management, and
project tooling, fully replacing Poetry.

## Context

Both [Poetry][poetry] and uv give fast, deterministic dependency resolution and
installs. They both also fully manage the creation and maintenance of virtual
environments. Out of the two, uv does have a speed advantage through is extensive use of
caching and native compilation.

One of the big advantages of uv is that in addition to managing project dependencies, it
can also manage the appropriate python runtime itself. Combined with the uvx tool that
comes with it, you now have a much simplified onboarding process for new starter on a
project. All they need installed is uv, and Docker to run the project up.

uv also works on Windows, something our previous [pyenv][pyenv] approach did not.

## Implications

Developers must install and use uv/uvx. We cannot support both the older Poetry with
Pyenv approach at the same time.

All pipelines, tasks and documentation also need to be updated to reference uv instead
of Poetry and pyenv.
